# 任务清单应用 - 规格说明书

## 1. 项目概述

- **项目名称**: TaskFlow - 任务进度管理器
- **类型**: 桌面应用 (Rust + Tauri)
- **核心功能**: 管理每日/每周/每月任务，跟踪学习进度，支持历史记录查询
- **目标用户**: 学生、自学者、需要任务管理的人群

## 2. UI/UX 规格

### 布局结构

- **主窗口**: 单一窗口应用，包含侧边栏 + 主内容区
- **侧边栏**: 导航菜单（任务类型切换）
- **主内容区**: 任务列表展示区域
- **底部**: 状态栏（显示统计信息）

### 响应式断点
- 最小窗口尺寸: 800x600px
- 侧边栏固定宽度: 220px

### 组件列表

1. **侧边导航**
   - 任务类型标签（全部/每日/每周/每月）
   - 历史记录入口
   - 未完成历史入口
   - 统计信息

2. **任务卡片**
   - 任务标题
   - 任务类型标签
   - 创建时间
   - 截止时间（可选）
   - 完成按钮
   - 删除按钮

3. **新建任务表单**
   - 任务标题输入
   - 任务类型选择
   - 截止日期选择（可选）
   - 提交按钮

4. **历史查询面板**
   - 时间类型选择（创建时间/截止时间）
   - 开始日期
   - 结束日期
   - 任务状态筛选（全部/已完成/未完成）
   - 查询按钮

5. **统计卡片**
   - 总任务数
   - 已完成数
   - 未完成数
   - 完成率

## 3. 功能规格

### 核心功能

1. **任务 CRUD**
   - 创建任务（标题、类型、截止日期）
   - 查看任务列表
   - 标记任务完成
   - 删除任务

2. **任务类型**
   - 每日任务 (daily)
   - 每周任务 (weekly)
   - 每月任务 (monthly)

3. **历史记录**
   - 查看已完成任务历史
   - 查看未完成任务历史
   - 按时间精确查询（某一天）
   - 按时间范围查询（开始日期 ~ 结束日期）

4. **数据存储**
   - SQLite 本地数据库
   - 任务持久化存储

### 数据模型

```rust
struct Task {
    id: i64,
    title: String,
    task_type: TaskType,  // daily, weekly, monthly
    created_at: DateTime,
    due_date: Option<DateTime>,
    completed: bool,
    completed_at: Option<DateTime>,
}
```

### 交互流程

1. 用户启动应用 → 显示今日任务列表
2. 点击侧边栏 → 切换任务类型视图
3. 点击"新建任务" → 弹出表单 → 提交保存
4. 点击任务完成按钮 → 标记为已完成
5. 点击"历史记录" → 进入历史查询页面
6. 选择日期范围 → 查询并显示结果
